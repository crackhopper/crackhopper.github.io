默认作用域（普通变量）

set(VAR value)
  • 作用域：当前 CMakeLists.txt 及当前目录的子目录 取决于 add_subdirectory 的顺序
  • 特性：
    • 子目录可以访问父目录的普通变量，但只有在 add_subdirectory() 之后才能看到
    • 父目录看不到子目录中 set 的普通变量

子目录作用域规则 add_subdirectory 创建 新的作用域

  • 子目录可以读取父目录已经 set 的普通变量
  • 子目录 set 的普通变量只在子目录有效（不会影响父目录）

函数和宏的局部作用域

function(my_func)
    set(VAR_IN_FUNC "func_value")
endfunction()

my_func()
message(STATUS "VAR_IN_FUNC = ${VAR_IN_FUNC}")
  • 函数内部 set 的变量默认 局部作用域
  • 要让父目录或外部可见，必须用 PARENT_SCOPE：

CACHE 变量（全局可见）

set(MY_VAR "cache_value" CACHE INTERNAL "说明")
  • 作用域：全局，所有子目录都可以访问
  • INTERNAL：不出现在 GUI，只在 CMake 内部可用
  • FORCE / STRING 等可选参数可覆盖或声明类型

总结

变量类型scope子目录可见父目录可见典型用途
普通 set()当前 CMakeLists可见（add_subdirectory 后）不可见子目录继承一些配置
set(… PARENT_SCOPE)向父目录传播可见可见函数内部或子目录设置参数
CACHE全局可见可见外部依赖、find_package、安装路径
target / target_link_libraries全局 target 名可见可见依赖关系、include path / link path 传播
